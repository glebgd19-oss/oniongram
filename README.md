# 🧅 Oniongram

Анонимный веб-мессенджер — гибрид Telegram и Discord с дизайном Liquid Glass (glassmorphism 2.0), тёмной темой и неоновыми акцентами.

**Стек:** React 18 + TypeScript + Vite + Tailwind CSS + Framer Motion.

> Полное описание функций и структуры проекта — в [GUIDE.md](./GUIDE.md).

---

## 🚀 Быстрый старт (локально)

Требования: **Node.js 18+** (рекомендуется 20) и npm.

```bash
# 1. Клонируйте репозиторий
git clone https://github.com/<your-username>/oniongram.git
cd oniongram

# 2. Установите зависимости
npm install

# 3. Запустите dev-сервер
npm run dev
# → http://localhost:5173
```

Сборка продакшн-версии и локальный просмотр:

```bash
npm run build     # собирает в dist/
npm run preview   # локальный сервер для dist/
```

## 🌐 Деплой на GitHub Pages (бесплатно)

### Вариант 1 — GitHub Actions (рекомендуется, уже настроен)

В репозитории уже лежит workflow `.github/workflows/deploy.yml`.

1. Запуште проект в свой GitHub-репозиторий (ветка `main`).
2. В настройках репозитория: **Settings → Pages → Build and deployment → Source: GitHub Actions**.
3. Сделайте любой push в `main` — сайт соберётся и опубликуется на
   `https://<your-username>.github.io/oniongram/`.

В `vite.config.ts` указано `base: "./"`, поэтому сборка работает из любого подкаталога — имя репозитория менять не нужно.

### Вариант 2 — вручную через gh-pages

```bash
npm run build
npm run deploy   # публикует dist/ в ветку gh-pages
```

Затем в **Settings → Pages** выберите ветку `gh-pages` как источник.

## 🔑 Переменные окружения

Для демо-режима (mock-данные) переменные **не нужны** — проект работает полностью статически, без бэкенда.

Если захотите подключить настоящий бэкенд (рекомендуется **Supabase**, бесплатный тариф: Postgres + Auth + Realtime):

| Переменная | Описание |
| --- | --- |
| `VITE_SUPABASE_URL` | URL проекта Supabase (Project Settings → API) |
| `VITE_SUPABASE_ANON_KEY` | Публичный anon-ключ Supabase |

Локально: создайте файл `.env.local`:

```env
VITE_SUPABASE_URL=https://xxxx.supabase.co
VITE_SUPABASE_ANON_KEY=eyJ...
```

Для GitHub Actions: добавьте их в **Settings → Secrets and variables → Actions** — workflow уже пробрасывает их в сборку, если они заданы.

## ⚠️ Ограничения статического хостинга (GitHub Pages)

GitHub Pages — чистая статика: нет своего сервера и постоянных WebSocket-соединений. Поэтому:

- **В демо** нет ботов и фейкового контента: приложение начинается с чистого аккаунта (только «Избранное» — чат с самим собой). Отправленные сообщения живут до перезагрузки страницы. Реальные собеседники, каналы и сервера появятся после подключения бэкенда.
- **Для продакшн-realtime** подключите внешний сервис с бесплатным тарифом: фронт остаётся на Pages, а WebSocket держит сторонний сервер:
  - **Supabase Realtime** — Postgres + Auth + realtime-подписки (рекомендуется);
  - Firebase Firestore / Realtime Database;
  - Ably / Pusher (бесплатные квоты сообщений).
- **Звонки**: в проде — WebRTC (P2P, бесплатные публичные STUN-серверы), сигналинг — через Supabase Realtime. В демо звонок имитируется оверлеем CallWindow.
- **Аутентификация**: в демо — локальная (логин/пароль + капча + опциональная 2FA, без сервера). В проде — Supabase Auth без email: технический адрес вида `username@oniongram.local` хранится только как ключ, телефон и реальная почта не требуются.

## Релиз 1.1.0

- **Полностью убраны боты и фейковый контент**: нет вымышленных пользователей, чатов, каналов, серверов и «чужих устройств» в настройках.
- Приложение начинается с чистого аккаунта: ваш @username берётся из формы входа, из чатов есть только «Избранное» (личные заметки, как в Telegram).
- Добавлены аккуратные пустые состояния для чатов, каналов и серверов.
- Отправленные сообщения теперь не пропадают при переключении разделов (хранятся до перезагрузки страницы).
- Удалено автономное демо `demo/index.html` (содержало демонстрационные данные).
- Защита от ботов (капча при регистрации + rate-limit сообщений) сохранена как штатная функция.
