# 🧅 Oniongram — гайд по функциям и структуре проекта

Oniongram — анонимный веб-мессенджер, гибрид Telegram (личные чаты, каналы) и Discord (сервера с текстовыми/голосовыми каналами и ролями). Акцент — приватность: без телефона и email, скрытые статусы, шифрование.

---

## 1. Функции

### 🔐 Аутентификация и приватность
- **Регистрация только по логину и паролю** — никаких телефонов, email и персональных данных (`Auth.tsx`).
- **Уникальные @username** — 3–20 символов, `a-z`, `0-9`, `_` (`validateUsername` в `security.ts`). По ним ищут людей, каналы, сервера и звонят.
- **Индикатор надёжности пароля** — 4 уровня (`passwordStrength`).
- **Капча при регистрации** — арифметическая задача (`makeCaptcha`), защита от ботов.
- **Rate-limit отправки сообщений** — не более 5 сообщений за 10 секунд (`createRateLimiter`), антиспам.
- **2FA (опционально)** — переключатель при регистрации и в настройках приватности.
- **E2E-шифрование (имитация для демо)** — текст шифруется локально перед «отправкой» и расшифровывается при отображении (`demoEncrypt`/`demoDecrypt`, ключ сессии в `sessionStorage`). В проде заменяется на libsodium/WebCrypto.
- **Скрытые «последний визит» и статус онлайн** — тумблеры в Настройки → Приватность; статус «невидимка» 🕶️ у пользователей.

### 💬 Личные чаты (`ChatWindow.tsx`)
- Текстовые сообщения с индикатором 🔒 и галочками доставки.
- Фото (градиентные заглушки в демо), файлы (имя + размер), голосовые с анимированной волной.
- Реакции на сообщения, индикатор «печатает…» с анимированными точками.
- Антиспам: rate-limit на отправку + предупреждение.

### 📞 Звонки (`CallWindow.tsx`)
- Голосовые и видеозвонки из шапки чата, профиля или поиска по @username.
- Стеклянный оверлей с таймером, кнопками микрофон/камера/завершить и пульсирующим аватаром.
- В демо звонок имитируется; прод-путь: WebRTC + сигналинг через Supabase Realtime.

### 📡 Каналы (`ChannelView.tsx`) — как в Telegram
- Подписка/отписка, счётчик подписчиков, значок верификации.
- Посты с реакциями (кликабельные, счётчик растёт) и счётчиком просмотров 👁.

### 🏠 Сервера (`ServerView.tsx`) — как в Discord
- Текстовые (#general, #privacy…) и голосовые (🔊 Lounge) каналы; видно, кто сейчас в войсе.
- **Роли** с цветами (Admin, Moderator, Member…) и **списками прав** (управление каналами, бан, приглашения…).
- Панель участников, сгруппированная по ролям; бейджи ролей у сообщений; медленный режим (антиспам).

### 🔍 Глобальный поиск (`Search.tsx`)
- Горячая клавиша **⌘K / Ctrl+K**, закрытие по Esc.
- Ищет по пользователям (@username и имя), каналам, серверам и тексту сообщений.
- Прямо из результатов можно открыть чат или позвонить 📞.

### ✨ Premium — бесплатно для всех
- Кастомные статусы, увеличенный лимит файлов (2 ГБ), анимированные эмодзи/стикеры, кастомные темы.
- Оформлено как «premium», но активно у каждого аккаунта — никакой монетизации (вкладка Premium в Настройках).

### ⚙️ Настройки (`Settings.tsx`) — стиль Telegram + Discord
- **Приватность**: скрыть последний визит / онлайн, 2FA, E2E (всегда вкл.), блокировка пользователей.
- **Уведомления**: чаты / каналы / звук.
- **Оформление**: тёмная тема по умолчанию, выбор неонового акцента (фиолетовый/бирюзовый/маджента).
- **Устройства**: список сессий, завершение чужих сессий; IP не хранятся.
- **Данные**: экспорт всего архивом, автоудаление сообщений, удаление аккаунта.

### 👤 Профили (`ProfileCard.tsx`)
- Только @username, аватар-градиент, кастомный статус — никаких персональных данных.
- Кнопки звонка и видеозвонка прямо из карточки.

## 2. Дизайн-система Liquid Glass

Всё определено в `src/index.css` + `tailwind.config.js`:

| Класс | Назначение |
| --- | --- |
| `.glass` | Базовое стекло: `backdrop-filter: blur(20px) saturate(180%)`, фон rgba с alpha ~0.15–0.3, обводка 1px `rgba(255,255,255,.15)`, мягкая тень «парения» |
| `.glass-strong` | Более плотное стекло для модалок и плотных экранов (читаемость важнее эффекта) |
| `.glass-refract` | Цветное преломление по краям (градиентная рамка через mask-composite) |
| `.specular` | Блик, следующий за курсором (CSS-переменные `--spec-x/--spec-y`, обновляются в `main.tsx` через rAF) |
| `.bg-scene` + `.blob-*` | Анимированный градиентный фон с дрейфующими неоновыми пятнами (только transform — 60 FPS) |
| `.skeleton`, `.typing-dot`, `.dot-online`, `.wave-bar` | Skeleton-загрузка, тайпинг, пульс онлайн-статуса, волна голосового |

Принципы:
- **Слои по глубине**: ленты сообщений — обычный непрозрачный слой; сайдбар, шапки, модалки, звонки, поиск — стекло «парит» сверху.
- **Spring-анимации Framer Motion** — общий пресет `glassSpring` (~300–400 мс): переходы экранов, появление сообщений, hover/tap, `layoutId` для «перетекания» активных вкладок.
- **60 FPS**: анимируются только transform/opacity, блик — через rAF-троттлинг; есть поддержка `prefers-reduced-motion`.
- **1440p-first**: рассчитано на 2560×1440, адаптив вниз через breakpoints Tailwind (`max-lg`, `max-md`, `xl`).

## 4. Как устроены данные (mock)

`src/data/mock.ts` экспортирует типы и наборы данных:

- `users` — пользователи со статусами (`online/idle/dnd/hidden/offline`), флагами `premium` и `hideLastSeen`;
- `chats` — личные чаты с разными типами сообщений (`text/photo/file/voice`), реакциями и тайпингом;
- `channels` — каналы с постами, просмотрами и подписчиками;
- `servers` — сервера с ролями (цвет + список прав), каналами и сообщениями;
- `premiumPerks`, `devices` — данные для настроек.

Чтобы подключить реальный бэкенд, замените импорты из `data/mock.ts` на запросы к Supabase (таблицы `users`, `chats`, `messages`, `channels`, `servers` + Realtime-подписки) — интерфейсы типов уже готовы.

## 6. Демо-ограничения (честный список)

- Ботов и фейкового контента нет: приложение стартует с чистого аккаунта. Единственный чат — «Избранное» (заметки самому себе); реальные собеседники, каналы и сервера появятся после подключения бэкенда (Supabase Realtime).
- Сообщения/звонки/аккаунты не сохраняются между перезагрузками — нет сервера (GitHub Pages = статика).
- E2E-шифрование — демонстрационное (XOR + сессионный ключ), в проде — настоящая криптография (WebCrypto/libsodium, протокол типа Signal).
- Звонки имитируются UI-оверлеем; прод-путь — WebRTC + Supabase Realtime для сигналинга.
- Капча и rate-limit работают на клиенте; в проде их нужно дублировать на сервере (Supabase RLS + edge functions).
